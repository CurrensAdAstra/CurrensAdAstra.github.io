# Teaching

제 강의 철학은 **이론적 컴퓨터공학 개념과 실제 시스템 구현의 연결**에 있습니다.  
학생은 추상 개념을 이해하는 것뿐 아니라, 실제 시스템에서 그 개념이 어떻게 동작하는지 체험할 때 가장 잘 학습합니다.

그래서 강의는 **실습 기반 시스템 개발, 디버깅, 성능 분석**을 중심으로 구성해 신뢰 가능한 소프트웨어를 설계하는 직관을 기르도록 합니다.

---

## Teaching Philosophy

공학 교육은 **시스템 레벨 사고**를 길러야 한다고 믿습니다.

학생은 개별 컴포넌트 동작뿐 아니라, 컴포넌트가 결합될 때 시스템이 어떻게 거동하는지 배워야 합니다. 이를 위해 다음을 강조합니다.

- 실습 중심 실험과 디버깅
- 시스템 아키텍처와 컴포넌트 상호작용 이해
- 학술 개념과 실무 엔지니어링 연결
- 실제 시스템을 만들고 개선하는 학습

단순 이론 설명보다 **실제 동작 관찰과 반복 개선 학습**을 지향합니다.

## Teaching Approach

강의는 보통 **개념 → 구현 → 시스템 분석** 순서로 진행합니다.

주요 활동:
- 라이브 코딩 포함 인터랙티브 강의
- 임베디드 개발 플랫폼 실습
- 실문제 기반 프로젝트 학습
- 가독성/유지보수성 중심 코드리뷰
- 프로파일링/디버깅 도구 기반 성능 분석

이 접근은 학생이 **개념 이해와 엔지니어링 직관**을 함께 갖추게 합니다.

## Teaching Goals

장기적으로는 **임베디드 시스템, 로보틱스 소프트웨어, 고성능 컴퓨팅**을 통합한 강의 체계를 구축하는 것이 목표입니다.

- Advanced Embedded Systems
- Robotics Software Development with ROS2
- GPU Programming and Parallel Computing
- Systems Performance Analysis

이는 현대 컴퓨팅 환경에서 커지는 **시스템 엔지니어링 역량**의 중요성을 반영합니다.
